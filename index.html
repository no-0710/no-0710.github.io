配置 Ganglia 监控 HBase

发表于 2017-08-25 | 分类于 HBase

Ganglia 简介

有兴趣的可以去看看其论文：

  • The ganglia distributed monitoring system: design, implementation, and experience
  • Wide area cluster monitoring with Ganglia

简介

由 UC Berkeley 发起的一个开源集群监视项目，设计用于测量和监控数以千计的节点。

主要是用来监控系统性能，如 CPU、内存、磁盘使用率，I/O 负载、网络流量情况等。

ganglia 采用多层次的结构模式，使得其可以实现良好的扩展。

gmond 带来的系统负载非常少，这使得它成为在集群中各台计算机上运行的程序，而不会影响用户性能。

组件

  • gmond 是一个守护进程，运行在每个需要监控的节点上，收集监控统计，发送和接受在同一个组播或是单播通道上的统计信息
  • gmetad 也是一个守护进程，它会定期检查 gmond，从那里拉取数据，并将其存储在 RRD 存储引擎中。其也可以查询多个集群并聚合信息。
  • ganglia-web 读取 RRD 文件，动态生成各类性能监控图表，并提供 web 服务。

HBase 管理工具

发表于 2017-08-24 | 分类于 HBase

本文档中的 HBase 版本为 hbase-1.2.6，由于版本的问题，其设置或是默认参数可能不一致，或是其他版本，请移步到 官方文档 查看。

Web 管理工具

Web 管理工具是 HBase 对外提供的一个 Web 接口，可以通过浏览器进行访问。建议使用 Google Chrome 兼容性会好些。

HMaster 的 Web 接口

  1. 端口参数：通过 hbase-site.xml 中的 hbase.master.info.port 进行修改，默认值为16010
  2. 通过 http://192.168.7.201:16010 访问

  1. HMaster Web 管理界面包含的状态信息
    • HBase 的版本信息
    • HBase 的基本配置信息
    • HBase在HDFS的存储路径
    • ZooKeeper的节点信息
    • 集群的负载信息
    • 表、region 和 region server 的信息
    • 可以进行 compat和 split 操作
